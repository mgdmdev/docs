## Overview

Applications in mgPass represent OAuth 2.0 clients that authenticate users. Each application gets a client ID and secret, and is configured with redirect URIs, allowed scopes, and token lifetimes.

## Application Types

| Type | Description | Has Secret | Example |
|------|-------------|-----------|---------|
| **Traditional Web** | Server-rendered apps with a backend | Yes | Next.js, Django, Rails |
| **SPA** | Single-page applications (use PKCE) | No | React, Vue, Angular |
| **Native / Mobile** | iOS, Android, desktop apps (use PKCE) | No | Swift, Kotlin, Flutter |
| **Machine-to-Machine** | Server-to-server with no user context | Yes | Cron jobs, microservices |

## Register an Application

<CodeGroup>
```bash cURL
curl -X POST https://pass.mediageneral.digital/api/clients \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "adesa+ Web",
    "type": "traditional_web",
    "redirect_uris": ["https://adesa.plus/callback"],
    "post_logout_redirect_uris": ["https://adesa.plus"],
    "allowed_scopes": ["openid", "profile", "email", "stream:live"],
    "cors_origins": [],
    "access_token_ttl": 3600,
    "refresh_token_ttl": 2592000
  }'
```

```javascript JavaScript
const response = await fetch("https://pass.mediageneral.digital/api/clients", {
  method: "POST",
  headers: {
    Authorization: "Bearer ADMIN_TOKEN",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "adesa+ Web",
    type: "traditional_web",
    redirect_uris: ["https://adesa.plus/callback"],
    post_logout_redirect_uris: ["https://adesa.plus"],
    allowed_scopes: ["openid", "profile", "email", "stream:live"],
    cors_origins: [],
    access_token_ttl: 3600,
    refresh_token_ttl: 2592000,
  }),
});
```
</CodeGroup>

**Response:**

```json
{
  "id": "app_abc123",
  "name": "adesa+ Web",
  "type": "traditional_web",
  "client_id": "app_abc123",
  "client_secret": "cs_live_xxxxxxxxxxxxxxxx",
  "redirect_uris": ["https://adesa.plus/callback"],
  "allowed_scopes": ["openid", "profile", "email", "stream:live"]
}
```

<Warning>
The `client_secret` is shown only once at creation time. Store it securely.
</Warning>

## Configuration Options

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Display name shown on consent screen |
| `type` | string | `traditional_web`, `spa`, `native`, `machine_to_machine` |
| `redirect_uris` | string[] | Allowed OAuth callback URLs |
| `post_logout_redirect_uris` | string[] | Allowed post-logout redirect URLs |
| `allowed_scopes` | string[] | Scopes this app can request |
| `cors_origins` | string[] | Allowed CORS origins (SPA only) |
| `access_token_ttl` | integer | Access token lifetime in seconds (default: 3600) |
| `refresh_token_ttl` | integer | Refresh token lifetime in seconds (default: 2592000) |
| `logo_uri` | string | App logo URL for consent screen |
| `description` | string | App description for consent screen |

## Per-App Branding

Override the default sign-in screen branding for specific apps:

```json
{
  "branding": {
    "logo_uri": "https://adesa.plus/logo.png",
    "primary_color": "#FF6B00",
    "company_name": "adesa+"
  }
}
```

## Secret Rotation

Rotate a client secret without downtime. The old secret remains valid for a grace period.

```bash
curl -X POST https://pass.mediageneral.digital/api/clients/app_abc123/rotate-secret \
  -H "Authorization: Bearer ADMIN_TOKEN"
```

**Response:**

```json
{
  "client_secret": "cs_live_new_xxxxxxxxxxxxxxxx"
}
```

## Update an Application

```bash
curl -X PATCH https://pass.mediageneral.digital/api/clients/app_abc123 \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "redirect_uris": [
      "https://adesa.plus/callback",
      "https://staging.adesa.plus/callback"
    ]
  }'
```

## Delete an Application

```bash
curl -X DELETE https://pass.mediageneral.digital/api/clients/app_abc123 \
  -H "Authorization: Bearer ADMIN_TOKEN"
```

<Warning>
Deleting an application immediately invalidates all tokens issued to it. Users will need to re-authenticate with a different application.
</Warning>
